# DD-002 - Centralized Service Instance Creation

## Problem

A decision had to be made regarding where service instances should be created.

## Alternatives Considered

### Option A

Each React component creates its own service instance.

### Option B

Each service exports a global singleton instance.

### Option C (Chosen)

Create all service instances from a single location and reuse them throughout the application.

## Decision

Service instances are created centrally and then imported wherever they are needed.

## Rationale

- Provide a single configuration point.
- Avoid duplicated initialization code.
- Keep React components focused exclusively on presentation logic.

## Consequences

### Advantages

- Centralized configuration.
- Lower coupling.
- Cleaner components.
- Changes to service initialization only require modifying a single file.

### Drawbacks

- Adds one additional level of indirection when following the application's execution flow.
