# DD-003 - Stateless React Application

## Problem

A decision had to be made regarding how data retrieved from the REST API should be managed.

## Alternatives Considered

### Option A

Maintain a global application state using a store.

### Option B (Chosen)

Each page loads the data it requires directly from the REST API.

## Decision

Each page performs its own data loading using `useEffect`, without maintaining shared application state.

## Rationale

- Simpler architecture.
- Lower implementation complexity.
- Data is always up to date.
- Follows the approach recommended during the SPA course for small REST-based applications.

## Consequences

### Advantages

- No state synchronization issues.
- Straightforward implementation.
- Always displays fresh data.

### Drawbacks

- Some API requests may be repeated when navigating between pages.
